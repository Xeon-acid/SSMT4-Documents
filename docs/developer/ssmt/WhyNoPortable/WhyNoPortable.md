# 为什么不设计成解压即用？

以Portable发布文件？解压即用的打包好的工具？

嗯，因为Rust + Tauri架构默认的打包工具就只有两个选择，要么使用nsis打包为exe的安装包

要么使用wix打包为.msi的安装包

SSMT4选择只使用nsis打包为exe的安装包，原因如下：

- .msi的打包有bug，部分情况下无法成功打包
- wix已停更很久，nsis还在更新
- 如果打包为Portable，则可能被倒卖的人做成整合包

恭喜你发现了另一个彩蛋，那就是为什么SSMT要设计成先进行配置后才能正常使用这样子。

因为如果设计成非常简单的解压即用，那么会导致大量不喜欢思考的人能够轻松使用。

通过一些配置进行拦截，可以精准过滤出Mod作者，或者对Mod有热爱的或者至少有一定理解的用户出来。

至少要知道如何复杂的完成简单的操作，也就是得稍微了解一点3Dmigoto是干啥的。

也就是说，这个工具是设计给Mod作者使用的。

这样就防止了Mod使用者也可以轻松的使用。

为什么这么设计呢，那就不得不说d3dxSkinManager被倒狗做成整合包，疯狂传播。

甚至最后使用d3dxSkinManager的官方群作为售后群的奇葩行为，最终导致d3dxSkinManager被原神直接拉入进程黑名单的故事了。

也就是说，本工具的初始配置阶段，具有一定的防自学设计，如果有人想要使用本工具去发布Mod整合包

那么最低也要教会用户如何正确使用本工具。

这在Mod倒卖领域是行不通的，因为倒卖面向的都是懒人，懒到不想进行任何复杂操作，开箱即用的人。

为了防止SSMT被人作为倒卖Mod的整合包中的启动工具，SSMT特地设计了以下防自学设计：

1.用户拿到手之后，必须连接到Github才能下载到对应的3Dmigoto安装包，这一步可以卡死99%的没有魔法的用户

正是这99%的没有魔法的用户，他们信息受限，也懒得折腾，才会去买倒卖的整合包。

2.用户拿到手之后，必须手动配置游戏预设为指定的游戏配置，且这个游戏预设名称不是简单的 原神、崩坏三 等等

这样直观的名称，而是 GIMI、SRMI、WWMI这种Mod工具的代称，如果你不了解Mod圈子的常见工具

你甚至不知道这是做什么的，那么你使用了错误的游戏预设，即使你有梯子，你也会下载到错误的Package包导致Mod无法正常加载

没错，这一步精准过滤了对Mod制作和Mod加载器包并不了解的人。

剩下的人里大概率是作者了。

3.SSMT缓存文件夹设计

又来了一手阴间设计，这个目录默认是设置到C盘的AppDataLocal下面的，对于大部分普通人来说，C盘的空间都是不够用的

即使是Mod作者，大部分也需要手动把这个目录改到其他的地方，才能确保不会让dump把你的磁盘塞满。

这一个设计就能拦住部分喜欢开箱即用的懒人，如果连这个都不会自己设置的话，我很怀疑他能否学会Mod制作。

同时这也是一个防自学设计，也是一个过滤器

三层过滤下来，那就剩不下多少人了，对吧，没错要的就是这个效果。

4.配置文件无法随程序迁移

也就是说，配置文件不是放在程序安装目录下的，而是放在全局的AppDataLocal里的，这就导致了你不能提前配置好SSMT

然后把配置好的SSMT直接分发给用户，确保用户必须走一遍配置流程，也是过滤器

5.3Dmigto目录可以自由选择

没错，不会在程序中自己附带3Dmigoto包，而是让需要的用户去手动选择目录，这样不仅非常灵活

而且也避免了其他人把3Dmigoto和SSMT塞到一起，发到用户手里的问题

# 层层过滤，精准避免工具被倒卖

嗯，有人就要说了，为什么不能做成开箱即用呢，那必然是不可能开箱即用的。

- 国内不比国外，人火了，或者工具火了是要出问题的，例如d3dxSkinManager、胡桃工具箱、狩野樱。
- 过滤掉倒卖工具和不愿意进行任何额外配置的人。
- 限制圈子膨胀，提高圈子质量

哦，那你要说了，有人会去做教程，教用户如何使用，你怎么防呢？

- 会的不用教，不会的教也教不会，大部分人都是懒人不可能为了玩Mod专门去学复杂的工具配置，即使它并不复杂

但是现代化社会，抖音上视频2秒内不好看都会划走，何况一个需要5分钟才能讲清楚的配置

- 做视频出教学，反而会精准过滤出那些有耐心，愿意付出时间学习的高质量人类，这些人大部分不会去买倒卖的东西

这个是某种心理现象吧，如果一个人认为自己懂得比别人多，那么他就会把自己的定位和其他人划分开来

此时就不会做大部分人都会做的事情，才能赋予自身独特的认知感，才能维持自己的San值。

- 问题本身就存在争议，别人为什么不用开箱即用的Mod管理器呢？

现在同类型的管理器，少说也有50多款，大部分都是开箱即用的，

也能确保使用SSMT的用户是被精准过滤的


# 最终效果

SSMT的用户被划分为一个很小的圈子，风险低，质量高。

当然上面的是随手写的，可能原因描述的不够详细，总之是被需求驱动不得不这样设计。

不是做不到解压即用，而是不能做到解压即用。
